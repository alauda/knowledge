# 如何为 ACP 集群添加健康检查

## 概述

ALB 健康检查端口是每个集群中预安装的，用于监控集群健康状态的端点。如果此端口无法访问，则表示集群处于不健康状态。

## 先决条件

在 VIP 基于访问配置中利用 ALB 健康检查端口进行集群健康监控时，必须在虚拟 IP (VIP) 上为指定的健康检查端口设置端口转发规则。如果未配置此端口转发，将导致健康检查无法正常工作。

## 配置 VIP 健康检查

以下表格指定了虚拟 IP (VIP) 健康检查实施所需的配置参数：

| 健康检查参数          | 描述                                                                                                                                                            |
| --------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 地址                   | ALB 的地址                                                                                                                                                     |
| 端口                   | 11782                                                                                                                                                          |
| 协议                   | 健康检查的协议类型，建议使用 TCP。                                                                                                                             |
| 响应超时              | 接收健康检查响应所需的时间，建议配置为 2 秒。                                                                                                                  |
| 检查间隔              | 健康检查的时间间隔，建议配置为 5 秒。                                                                                                                          |
| 不健康阈值            | 确定后端服务器健康检查状态为失败的连续失败次数，建议配置为 3 次。                                                                                             |
