# 排查 Redis 集群模式连接错误

## 介绍

连接到运行在 **集群模式** 下的 Alauda Cache Service for Redis OSS 实例的应用程序可能会出现以下错误：

| 症状                                                   | 含义                                                                                                                                      |
| ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| `ERR SELECT is not allowed in cluster mode`           | 客户端调用了 `SELECT n`（数据库切换）。集群模式仅支持数据库 `0`。                                                                          |
| `MOVED <slot> <host>:6379`                            | 客户端连接到一个分片，但请求的键位于不同的分片。当前连接未跟随集群重定向。                                                                  |
| `unable to connect` 在看似正确的配置后              | 仅配置了一个种子节点，而该节点无法访问，**或** 客户端使用了非集群 API。                                                                     |

这三种情况都指向相同的根本原因：**使用了非集群（独立或哨兵）客户端与 Redis 集群进行通信。** 本文档重点在于诊断症状并将客户端切换到集群 API。它并不涵盖如何从头开始编写集群感知客户端 — 有关完整的客户端配置示例，请参阅公共文档：

- [如何访问集群实例](https://docs.alauda.io/redis/5.0/how_to/access/20-cluster.html)（涵盖 Jedis、Lettuce、Redisson、go-redis）

## 前提条件

- 部署在 **集群** 模式下的 Alauda Cache Service for Redis OSS 实例。
- 具有上述症状之一的失败客户端应用程序。
- 实例的 **访问方式** 页面上的集群节点端点列表。

## 诊断

### 1. 确认实例处于集群模式

在平台 UI 中，打开 Redis 实例详细信息页面，验证 **架构** 是否为 `cluster`（而不是 `sentinel` 或 `standalone`）。如果实例为 sentinel/standalone，则上述症状不是原因 — 请调查实际错误消息。

### 2. 检查使用的客户端库 API

检查客户端代码或框架配置：

| 症状              | 查找内容                                                                                     | 修复                                                                   |
| ------------------ | ------------------------------------------------------------------------------------------- | --------------------------------------------------------------------- |
| `SELECT not allowed` | `SELECT 1`、`spring.redis.database: 1`、`Jedis#select(int)`                                 | 移除数据库切换调用；集群模式仅支持数据库 `0`。                          |
| `MOVED ...`        | `Jedis`（单节点）、`RedisClient`（Lettuce，单端点）、`redis.client: standalone`          | 切换到集群 API：`JedisCluster`、`RedisClusterClient` 等。              |
| 单主机配置         | `spring.redis.host`、`redis.host`                                                           | 替换为列出多个种子节点的集群模式块。                                   |

### 3. 检查种子节点列表

集群模式通过与您提供的种子节点进行通信来发现拓扑。如果仅配置了一个种子节点，而该 Pod 被重新调度或重启，客户端可能会在启动时失败。**始终列出至少三个主端点**，以便在单节点故障时引导成功。

## 解决方案

1. 从应用程序中移除所有 `SELECT n`（或等效的多数据库选项）。
2. 将客户端切换到其 **集群 API** — 请参阅上述公共文档以获取每个库的确切代码。
3. 提供 **所有主端点**（或至少三个）作为种子节点列表。
4. 确保每个分片上的密码相同 — 集群中的所有节点共享相同的密码；使用平台提供的默认用户密钥。
5. 重启应用程序并查看日志以确认 `MOVED` / `SELECT` 错误已消失。

## 重要注意事项

- **通过添加更多主机条目无法使独立客户端与集群模式一起工作** — 必须使用集群 API。没有等效的适配器。
- **跨槽的管道不受支持。** 如果您的应用程序对多个键进行管道处理，请使用哈希标签（`{tag}`）将它们分组到同一槽中 — 例如，`user:{42}:profile` 和 `user:{42}:sessions` 都哈希到 `42`。
- **外部访问。** 当客户端在 Kubernetes 集群外部运行时，每个分片的主节点和副本必须可访问。通过 NodePort 或 LoadBalancer 暴露它们，并在种子列表中使用这些地址。
- **操作员行为：** 操作员不会将独立连接转换为集群连接。如果出现 `MOVED` 错误，责任在客户端。
