# 如何查看 Redis 慢日志

## 介绍

本指南描述了如何检查、管理和配置 Redis 慢日志。慢日志记录超过可配置执行时间阈值的命令，是诊断性能问题和识别低效查询的关键工具。

## 前提条件

- 通过 `redis-cli` 或其他 Redis 客户端访问 Redis 实例。
- 身份验证凭据（如果实例启用了密码身份验证）。

## 操作步骤

### 1. 连接到 Redis

使用 `redis-cli` 进入 Redis shell：

```bash
redis-cli -h <redis-host> -p <redis-port> -a <password>
```

### 2. 查看慢日志条目数量

检查当前存储在慢日志中的条目数量：

```text
SLOWLOG LEN
```

### 3. 查看慢日志条目

检索最新的慢日志条目。参数指定要返回的条目数量：

```text
SLOWLOG GET <count>
```

示例输出：

```text
redis 127.0.0.1:6379> SLOWLOG GET 2
1) 1) (integer) 14            // 唯一条目标识符
   2) (integer) 1309448221    // 命令执行的 Unix 时间戳
   3) (integer) 15            // 执行时间（微秒）(15 μs)
   4) 1) "ping"               // 命令及其参数
2) 1) (integer) 13
   2) (integer) 1309448128
   3) (integer) 30            // 执行时间（微秒）(30 μs)
   4) 1) "slowlog"
      2) "get"
      3) "100"
```

每个条目包含：

| 字段    | 描述                                             |
| ------- | ------------------------------------------------ |
| 索引 1  | 慢日志条目的唯一标识符                          |
| 索引 2  | 命令执行时的 Unix 时间戳                        |
| 索引 3  | 执行命令所花费的时间（以微秒为单位）            |
| 索引 4  | 命令及其参数                                    |

### 4. 重置慢日志

清除所有慢日志条目：

```text
SLOWLOG RESET
```

## 配置参数

使用以下 Redis 参数配置慢日志行为。这些参数可以通过实例的 `customConfig` 设置，或在运行时使用 `CONFIG SET` 命令设置：

| 参数                       | 描述                                                                                                   | 备注                                                                                                   |
| ------------------------- | ----------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| `slowlog-log-slower-than` | 记录为慢日志条目的命令的最小执行时间（以微秒为单位）。                                                | 设置为 `0` 以记录每个命令。设置为 `-1` 以禁用慢日志记录。                                             |
| `slowlog-max-len`         | 在慢日志中保留的最大条目数量。当达到限制时，最旧的条目将被删除。                                    | 默认值：`0`（不保留条目）。设置为正值，例如 `128` 或 `1024` 以供生产使用。                           |

示例：将阈值设置为 1 毫秒（`1000` 微秒），并保留最多 1024 条目：

```text
CONFIG SET slowlog-log-slower-than 1000   # 1000 微秒 = 1 毫秒
CONFIG SET slowlog-max-len 1024
```

要在重启后保留这些设置，请在实例的 `customConfig` 中进行配置：

```yaml
spec:
  customConfig:
    slowlog-log-slower-than: "1000"
    slowlog-max-len: "1024"
```

## 重要考虑事项

- 慢日志存储在内存中，并在实例重启时清除，除非通过监控堆栈持久化。
- 将 `slowlog-log-slower-than` 设置为 `0` 会记录每个命令，可能会对性能产生明显影响。仅在短期诊断窗口中使用。
- 增加 `slowlog-max-len` 会消耗额外的内存。选择适合您实例大小的值。
- 对于长期分析，定期抓取 `SLOWLOG GET` 输出并将其转发到集中式日志记录或 APM 系统。
