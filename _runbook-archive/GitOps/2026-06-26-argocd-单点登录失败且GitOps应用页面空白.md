# argocd 单点登录失败且 GitOps 应用页面空白

## 现象
- `argocd` 单点登录失败。
- 容器平台中 `GitOps 应用` 创建页面前端展示空白，无法正常使用。
- 问题在 `ACP 4.1.3` 环境中可复现，而内部 `4.2.1` 环境未出现相同现象，说明故障与平台版本和 GitOps 包版本组合有关。

## 排查过程与命令
- 首先使用 `argocd 4.3.0` 直接部署，发现单点登录不可用，且内部环境可以复现相同问题，说明问题并非单一客户环境脏数据导致。
- 随后在 `ACP 4.1.3` 环境中继续观察，发现容器平台 `GitOps 应用` 前端页面也不显示数据。结合内部 `4.2.1` 环境未复现这一事实，可以将排查方向收敛到平台版本与 GitOps 组件版本兼容性。
- 进一步确认后发现，`ACP 4.1.3` 实际并未适配 `gitops 4.3.0`，说明当前上传到 `ac` 的相关安装包版本存在错误。
- 为验证这一判断，重新上传 `gitops 4.2`，并在 `ACP 4.1.3` 环境中部署 `argocd-operator 4.2.0 + gitops 插件 4.2.0`。处理后，单点登录问题恢复，说明登录链路异常确实与版本不匹配有关。
- 针对 `GitOps 应用` 页面空白问题，继续回退并验证 `gitops 4.1` 版本。安装 `gitops 插件 4.1.8 + argocd-operator 4.2.0` 后，页面使用恢复正常。
- 综合上述验证结果，可以确认故障并非前端单独异常或 SSO 配置单独失效，而是 `ACP 4.1.3` 环境中安装了不匹配的 GitOps 相关版本包，导致单点登录和页面展示同时异常。

## 根因与修复方案
- **根因**
  - 上传到 `ac` 的 `argocd + gitops` 包版本与平台版本不匹配。
  - `ACP 4.1.3` 环境应使用 `gitops 4.1.x` 版本，但实际安装了不兼容版本，最终导致 `argocd` 单点登录失败以及 `GitOps 应用` 页面空白。
- **临时缓解方案**
  - 回退并重新安装与平台版本匹配的 GitOps 组件版本。
  - 在 `ACP 4.1.3` 环境中使用 `gitops 插件 4.1.8 + argocd-operator 4.2.0` 组合恢复功能。
- **根本解决方案**
  - 建立 `ACP` 与 `argocd/gitops` 安装包版本的兼容性校验机制，避免不匹配版本被上传到 `ac` 并进入安装链路。
  - 在发布和上传流程中固化版本矩阵检查，确保不同平台版本只能选择经过适配验证的 GitOps 组件包。
